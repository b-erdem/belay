# Formal verification layer

This directory holds a TLA+ model of Capstan's durable-execution core —
journaling, budget enforcement, cooperative cancellation, crash/reclaim,
workflow settlement — checked exhaustively with TLC, plus the execution
traces that ground it and the mutation specs that prove it bites.

The model was generated by [attest](https://github.com/b-erdem/attest)
(spec-from-source-and-traces) against `lib/capstan/runner.ex`,
`lib/capstan/storage/logic.ex`, `lib/capstan/storage/memory.ex`, and
`SCHEMA.md`, grounded in the traces under `traces/`. TLC then explored
**1,525,104 distinct states (depth 34) with zero violations** of:

| Property | Meaning |
|---|---|
| `NoExecutionPastBudget` | No step body runs once durable spend exceeds the budget — at most the single crossing step overshoots. |
| `CancelRequestSurvivesUntilTerminal` | A pending `cancel_requested` is never cleared by any transition (crash, reclaim, retry included). |
| `StepsJournaledAtMostOnce` | Memoized replay never re-executes a journaled step. |
| `TerminalStatesAreFinal` | Terminal states are exited only through the operator-retry gate (failed/cancelled → ready, or held for workflow members, with any pending cancel cleared). |
| `DepGate` | A running job's dependencies are satisfied — no retry path can put a dependent to work beside a failed dependency. |
| `TypeOK` | State-space sanity. |

## Why you should believe the model (mutation validation)

A green model is only meaningful if it would have caught real bugs. Both
production bugs found by Capstan's other verification layers are encoded
as mutations under `spec/mutations/`, each reverting the model to the
pre-fix semantics:

- **`PreFixBudget.tla`** — removes the pre-flight budget check (the bug
  the 7h endurance soak caught: 6 of 1,750 budget jobs paid for an extra
  step). TLC violates `NoExecutionPastBudget` with a counterexample that
  is step-for-step the soak's finding:
  `Insert → Claim → Exec b1..b3 (crossing step journaled) → Crash →
  Reclaim → Claim (attempt 2) → Exec b4`.
- **`PreFixCancelClear.tla`** — reclaim clears `cancel_requested` (the
  bug the adapter-equivalence property test caught in the Postgres ack
  path). TLC violates `CancelRequestSurvivesUntilTerminal` at exactly
  `CancelRunning → Crash → Reclaim`.
- **`PreFixRetry.tla`** — the pre-fix operator retry (straight to `ready`,
  pending cancel kept — bugs #3/#4, found by extending this model and then
  confirmed against the real engine). TLC violates `TerminalStatesAreFinal`
  in four states: `Insert → CancelParked → Retry` sends a dep-bearing job
  to `ready` instead of re-holding it.

The soak needed 99,004 jobs and 4,978 kills to hit the budget window six
times; TLC finds it deterministically in seconds. That is the point of
this layer: the chaos soak samples the state space, the model checker
exhausts it (within the modeled abstraction).

## What the model deliberately leaves out

The abstractions are recorded in `spec/Spec.tla`'s header and
`spec/FINDING.md`. Notably: exec-and-journal is one atomic action, so the
*at-least-once duplicate-execution window* (a step body re-running after a
crash between side effect and journal write) is outside this model — that
behavior is measured by the soak's at-least-once accounting instead.
Signals, awaiting, rate limiting, cron, uniqueness, and encryption are
also out of scope for these properties.

## Files

- `traces.exs` — deterministic scenario driver (Memory adapter, simulated
  clock, real claim/reclaim paths). Regenerate traces with
  `mix run verify/traces.exs`.
- `traces/*.jsonl` — seven grounding traces: happy path, memoized retry,
  exact budget kill, the budget crash window, cancel mid-run, cancel
  across crash/reclaim, workflow cascade-cancel.
- `spec/Spec.tla` + `Spec.cfg` — the canonical model (as generated;
  green).
- `spec/FINDING.md` — attest's structured report.
- `spec/mutations/` — the two pre-fix mutants (must fail TLC).
- `wake_protocol/` — a Lockstep controlled-concurrency model of the
  parent-wake protocol (race R1): the rc.1 count-gated variant loses the
  wake on a found, saved, replayable schedule; the shipped unconditional
  variant survives every explored interleaving. See its README.

## Re-running

```bash
# canonical (expect: no violations)
cd verify/spec
java -cp <tla2tools.jar> tlc2.TLC -workers 4 -config Spec.cfg Spec.tla

# mutations (expect: violations with the traces described above)
cd mutations
java -cp <tla2tools.jar> tlc2.TLC -workers 4 -config PreFixBudget.cfg PreFixBudget.tla
java -cp <tla2tools.jar> tlc2.TLC -workers 4 -config PreFixCancelClear.cfg PreFixCancelClear.tla
```

`tla2tools.jar` is the standard TLA+ toolbox jar (Java 17+). If the
engine's core semantics change, regenerate traces, re-run attest (or
update `Spec.tla` by hand), and keep the mutations failing.
